## Decisiones técnicas tomadas

### 1. Campo opcional en edición de tasas
**Decisión:** El campo `boss_commission_default` es OPCIONAL en edit, REQUERIDO en create.

**Razón:** 
- En create: siempre se debe establecer una comisión inicial
- En edit: permite actualizar solo tasas sin forzar cambio de comisiones
- UX más flexible y explícita

**Implementación:**
```php
// En create
'boss_commission_default' => 'required|numeric|min:0|max:100'

// En edit  
'boss_commission_default' => 'nullable|numeric|min:0|max:100'

// Solo actualiza si fue enviado
if ($request->filled('boss_commission_default')) {
    Seller::query()->update(['boss_commission' => ...]);
}
```

### 2. No usar promedio de comisiones
**Decisión:** No usar `avg('boss_commission')` como valor por defecto.

**Razón:**
- El promedio no representa ninguna "comisión global real"
- Si hay overrides, el promedio pierde significado
- Mejor mostrar distribución completa y dejar campo vacío u opcional

**Implementación:**
```php
// En edit: mostrar distribución, no promedio
$commissionGroups = Seller::select('boss_commission')
    ->groupBy('boss_commission')
    ->selectRaw('boss_commission, count(*) as count')
    ->get();

// Campo opcional con placeholder
placeholder="Dejar vacío para no cambiar"
```

### 3. Snapshots de tasas agregados
**Decisión:** Agregar snapshots de tasas (USD, EUR, VES) además de comisiones.

**Razón:**
- Trazabilidad completa para auditoría
- Permite saber qué tasas se usaron en cada venta
- Útil para reportes históricos y reconciliación

**Campos:**
- `usd_rate_snapshot` - Tasa BCV USD al momento de la venta
- `eur_rate_snapshot` - Tasa BCV EUR al momento de la venta
- `ves_rate_snapshot` - Tasa VES/PEN al momento de la venta

### 4. Comisión NO se guarda en exchange_rates
**Decisión:** La comisión del dueño aparece en el formulario de tasas pero NO se guarda en la tabla `exchange_rates`.

**Razón:**
- Evita redundancia (ya está en cada seller)
- Permite overrides por vendedor
- Formulario de tasas es solo una interfaz centralizada
- Actualización masiva directa en tabla `sellers`
