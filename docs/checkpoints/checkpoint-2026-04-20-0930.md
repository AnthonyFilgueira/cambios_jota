## Decisiones técnicas tomadas

### 1. Sistema de notificaciones con Mailpit
- Queue: database driver
- Listener: `ShouldQueue` para procesamiento asíncrono
- Email enviado al vendedor (temporal, hasta implementar REQ 2 completo)
- Mailpit captura emails en desarrollo: http://localhost:8025

**Producción:**
- Configurar SMTP en `.env`
- Cron job: `* * * * * cd /ruta && php artisan queue:work --stop-when-empty >/dev/null 2>&1`

### 2. Historicidad de datos
- **Snapshots**: Cada venta guarda comisiones en el momento exacto
- **Protección**: No se pueden editar tasas/comisiones si tienen registros asociados
- **Beneficio**: Reportes financieros exactos, auditoría completa

### 3. Fix crítico: Sale::observe() → markAsObserved()
- **Problema**: Conflicto con método estático `Model::observe()`
- **Solución**: Renombrado a `markAsObserved()`
- **Archivos afectados**: Sale.php, SaleController.php

### 4. Paleta Cambio J consistente
- Observaciones: naranja (alerta)
- Completada: teal (éxito)
- Aprobación: teal
- Rechazo: pink
