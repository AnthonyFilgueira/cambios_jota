## Decisiones técnicas tomadas

### 1. Validación en Dos Capas

**Controlador:**
- Validaciones con mensajes en español
- Orientado a errores de usuario
- `currency_pair_id` requerido
- Tasas > 0 (usd: 0.01+, eur: 0.01+, ves: 0.00001+)

**Modelo (boot):**
- Throw exceptions si valores <= 0
- Previene manipulación directa desde código
- Seguridad adicional

### 2. Filtro por Defecto: Solo Activas

**Razón:** No confundir al usuario con tasas históricas.

**Implementación:**
```php
if (!$request->filled('status')) {
    $query->where('is_active', true);
}
```

### 3. Activación Automática al Crear

**Razón:** UX simplificada, siempre se activa la nueva tasa.

**Implementación:**
```php
$rate = ExchangeRate::create(...);
$rate->activate(); // Desactiva las demás del mismo par
```

### 4. Evento `deleting` vs `deleted`

**Problema:** `deleted` causa error FK constraint.

**Solución:** Usar `deleting` (ANTES de eliminar).
